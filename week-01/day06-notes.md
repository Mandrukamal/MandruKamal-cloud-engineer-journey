# Security

## The shared responsibility model

In the shared responsibility model, the cloud provider is responsible for the security of the cloud infrastructure, while the customer is responsible for the security of their applications and data.

![Shared Responsibility Model](https://d1.awsstatic.com/onedam/marketing-channels/website/aws/en_US/product-categories/security-identity-compliance/compliance/approved/images/7a404923-5572-409c-b30e-6d44706bcd89.094727e5c591e9a96edf10578d0bc1172d9e4553.jpeg)

## Customer Responsibility
Customers are responsible for securing their applications and data, including:
Platform applications and identity and access management.
Operating system, network, and firewall configurations.
Client-side data encryption and data integrity authentication.
Server-side encryption and data protection.
Network traffic protection and monitoring.

## AWS responsibility
AWS is responsible for securing the underlying cloud infrastructure, including:
AWS Software: Compute, Storage, Database, and Networking
Hardware and AWS Global Infrastructure: Regions, Availability Zones, and Edge Locations

# When running the workloads in AWS, There are few tools:

1. Identity and Access Management (IAM)
2. Network and application protection
3. Data encryption and key management
4. Security monitoring and logging
5. Compliance and governance

## Different AWS Services for Security

1. AWS Identity and Access Management (IAM) - Manage user access and permissions
2. AWS Key Management Service (KMS) - Create and control encryption keys
3. AWS Artifact - Access compliance reports and security documentation
4. AWS Config - Track resource configurations and changes
5. AWS Shield - DDoS protection for applications

# AWS Trust Advisor

AWS Trusted Advisor is an online resource to help you reduce cost, increase performance, and improve security by optimizing your AWS environment. 
It provides real-time guidance to help you provision your resources following AWS best practices.

## Benefits of Trusted Advisor
1. Cost Optimization: Identify opportunities to reduce costs by eliminating unused or underutilized resources.
2. Performance Improvement: Get recommendations to improve the performance of your AWS environment.
3. Security Enhancements: Receive alerts about security vulnerabilities and best practices to secure your resources.
4. Fault Tolerance: Learn how to improve the fault tolerance of your applications by following AWS best practices.
5. Service Limits: Monitor your usage against AWS service limits to avoid service disruptions.
